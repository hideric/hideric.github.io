Kotlin的扩展库

Kotlin的扩展库

1.kotlin.coroutines（Kotlin协程库）

协程不一定是线程开启的，也有可能是进程开启的

通过提升CPU利用率，减少线程切换进而提升程序运行效率（不会像线程一样阻塞后占依旧用cpu）

协程的特性：

  1. 可控制：协程能做到可被控制的发起子任务
  2. 轻量级：协程非常小，占用资源比线程还少
  3. 语法糖：使多任务或多线程切换不再使用回调语法

启动协程的方式

  1. runBlocking : T // 用于执行协程任务，通常只用于启动最外层协程，用于从线程切换到协程
  2. launch : Job // 最常用的执行协程任务

  3. Async/await : Deferred // 用于执行协程任务，并得到执行结果

    fun main(args: Array<String>) = runBlocking<Unit> { // 外层携程
val job = launch { // 内层又开启协程
    repeat(1000) { i ->
        println("挂起中 $i ...")
        delay(500L)
    }
}
delay(1300L)
println("main:: 主线程等待中")
job.cancel()
job.join() // join函数不起作用，因为job协程已经被取消了
println("main:: 即将完成退出")
}
// 输出
挂起中 0 ...
挂起中 1 ...
挂起中 2 ...
main:: 主线程等待中
main:: 即将完成退出

    fun main(args: Array<String>) = runBlocking<Unit> {
val job = launch {
    repeat(1000) { i ->
        println("挂起中 $i ...")
        delay(500L)
    }
}
val job2 = async {
    delay(500L)
    return@async "hello"
}

println("job2的输出：" + job2.await())

delay(1300L)
println("main:: 主线程等待中")
job.cancel()
job.join()
println("main:: 即将完成退出")
}
// 输出
挂起中 0 ...
挂起中 1 ...
job2的输出：hello
挂起中 2 ...
挂起中 3 ...
main:: 主线程等待中
main:: 即将完成退出

launch方法分析

public fun CoroutineScope.launch(
    context: CoroutineContext = EmptyCoroutineContext, // 当前协程的上下文，通常用于协程间切换时传递参数；以及起到协程的调度器作用，将当前协程放到哪个线程中执行，被中断后在哪个线程恢复
    start: CoroutineStart = CoroutineStart.DEFAULT, // 当前协程以哪一种方式启动，默认当前线程什么时间空闲什么时间启动
    block: suspend CoroutineScope.() -> Unit // 真正执行的协程(Lambda闭包)
): Job {
    val newContext = newCoroutineContext(context)
    val coroutine = if (start.isLazy)
        LazyStandaloneCoroutine(newContext, block) else
        StandaloneCoroutine(newContext, active = true)
    coroutine.start(start, coroutine, block)
    return coroutine
}

协程的语法糖：允许用同步的方式发起一次异步的请求

fun displayDashboard(textView: TextView) = runBlocking {
		launch(UI) {
				val job = async(AndroidCommonPool) {
						// 不考虑异常的情况
						mOkHttpClient.newCall(mRequest).execute().body?.string()
				}
				textView.text = job.await()
		}
}

被suspend修饰的函数表示协程函数

// 被suspend修饰的函数只能被有suspend修饰的函数调用
// 因为suspend修饰的函数（或Lambda）被编译后会多一个参数类型叫Continuation
// 协程的异步调用本质上是一次回调
suspend fun getHtml(): String {
		return async(AndroidCommonPool) { URL("http://baidu.com").readText() }.await()
}

除上述三种启动协程的方式外，还有特殊的启动协程方式

  1. buildSequence/yield: Sequence // 用于执行会多次返回数据的协程任务

    val fibonacci = buildSequence {
   yield(1)
   var cur = 1
   var next = 1
   while (true) {
       yield(next)
       val tmp = cur + next
       cur = next
       next = tmp
   }
}
// yield 可以按需求调用，无限的返回数据
// yield 非常适合一些在无法预知终止条件的场景，比如直接读已经写好的本地文本，每次读一条
// 现在很多手机都有一种假装打电话的功能，你可以提前编辑好手机里要说的内容和每段内容的间隔
// 手机自动响起来，你就可以跟手机对话了，这种场景就非常适合。
fun main(args: Array<String>) {
   fibonacci.take(10).iterator().forEach {
       print("$it ")
   }
}

  2. buildIterator: Iterator

// 与buildSequence类似，只是返回值不同，由于Sequence本身就包含一个Iterator，所以通常不适用buildIterator

  1. produce: Channel // 用于执行协程任务，并得到一个channel（用于两个协程之间发送消息的管道）

    fun main(args: Array<String>) {
   val c = Channel<Int>()
   launch {
       get(c)
   }
   launch {
       put(c)
   }
   Unit
}
suspend fun get(channel: Channel<Int>) {
   while (true) {
       println(channel.receive())
   }
}
suspend fun put(channel: Channel<Int>) {
   var i = 0
   while (true) {
       channel.send(i++)
   }
}
