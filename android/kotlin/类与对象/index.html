类与对象

类与对象

1.构造函数

类声明方式:

class MainActivity : AppCompatActivity() // class 类名 : 父类,如果一个类的父类没有显式声明，那么父类是Any而不是Object，后面的父类的小括号代表调用父类的无参构造
class MainActivity : AppCompatActivity(), OnClickListener // 继承的接口可以直接在后面逗号隔开，不需要书写implements
// Kotlin的类默认是public final的
open class MainActivity : AppCompatActivity() // 如果不需要final，在类前面添加open修饰符

类的构造函数：

// MainActivity.kt
open class MainActivity(var int: Int) {
    init {
        println("xxxlog") // 类的构造函数被调用时执行
    }

    override fun hashCode(): Int {
        return super.hashCode()
    }

    override fun toString(): String {
        return super.toString()
    }
}
// 有多个构造函数的话需要显式得声明多级构造函数
class TestView : View {
  	// 次级构造函数必须直接或者间接的继承主构造函数或者父类构造函数
  	constructor(context: Context) : super(context) {
      	println("constructor")
    }
  	constructor(context: Context, attrs: AttributeSet?) : this(context, attrs, 0)
  	
  	constructor(context: Context, attrs: AttributeSet?, defStyleAttr: Int) : super(context, attrs, defstyleAttr)
}

2.访问修饰符

Kotlin中有4种访问修饰符：private、protected、public、internal

iternal是kotlin特有的，表示一个模块下的所有类都能访问到当前对象

3.伴生对象

Kotlin是没有静态方法的，解决方法有两种：

  1. @JvmStatic注解
  2. companion object伴生对象

companion object必须写在一个类内部

class StringUtils {
		companion object {
				fun isEmpty(str: String): Boolean {
						return "" == str
				}
		}
}
// 在kotlin中使用直接StringUtils.isEmpty()调用
// 在java中调用此方法
StringUtils.Companion.isEmpty("");

实际上伴生对象会在编译好以后在这个类生成一个静态对象，叫Companion对象，java在调用时是通过类的Companion对象去调用

4.单例类

伴生对象还有一个特性是用来生成单例

class Single private constructor() {
		companion object {
				fun get(): Single {
						return Holder.instance
				}
		}
		private object Holder {
				val instance = Single()
		}
}

5.动态代理

Kotlin在语言原生层面已经实现了动态代理

interface Animal {
    fun bark()
}

class Dog : Animal {
    override fun bark() {
        println("wang")
    }
}

class Zoo(animal: Animal) : Animal by animal { // 表示代理Animal
    override fun bark() {
        println("zoo")
    }
}

fun main() {
    Zoo(Dog()).bark()
}
// 当Zoo重写了bark()方法，不会再调用它代理的对象(Animal)的bark()方法，则输出"zoo"

kotlin会将动态代理编译后转化为静态代理，所以Kotlin的动态代理效率一定是比java动态代理效率高的(java通过反射，kotlin通过静态代理)

6.Kotlin特有的类

  1. 数据类

可以将类中的成员变量自动生成getter、setter方法，toString、hashCode、equals、copy等方法都写好了

只需要在类前面声明data关键字

   data class User(var id: Int, var name: String)

数据类是final类型的，不能再添加open去修饰它

   // 当使用一个类继承数据类时，提示This type is final
   class VIP(id: Int, name: String) : User(id, name)
  1. 枚举类

kotlin中枚举类使用与java中的一致，但是kotlin中一般使用密闭类代替枚举类

   sealed class SuperCommand {
   		object A : SuperCommand()
   		object B : SuperCommand()
   		object C : SuperCommand()
   		object D : SuperCommand()
   }
   fun exec(superCommand: SuperCommand) = when (superCommand) {
     	SuperCommand.A -> {}
     	SuperCommand.B -> {}
     	SuperCommand.C -> {}
     	SuperCommand.D -> {}
   }

密闭类是可以有子类的，但是子类必须和密闭类写在同一个文件中，通常会将密闭类的子类写在密闭类自身当中

密闭类最大的特点就是可以扩展它的子类，并且子类也可以成为密闭类的一个选项

   sealed class SuperCommand {
   		object UP : SuperCommand()
   		object DOWN : SuperCommand()
   		object LEFT : SuperCommand()
   		object RIGHT : SuperCommand()
     	class PACE(var pace: Int) : SuperCommand() // 扩展子类
   }
   fun exec(view: View, superCommand: SuperCommand) = when (superCommand) {
     	SuperCommand.UP -> {}
     	// ...
     	is SuperCommand.PACE -> {
       }
   }

数据类替代JavaBean；密闭类替代枚举类
